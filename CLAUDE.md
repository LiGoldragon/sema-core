# sema-core — rkyv Contract (askic↔semac)

sema-core defines every type that appears in the rkyv message
between askic and semac. corec generates Rust with rkyv derives
from the .aski definitions. Both askic (serializer) and semac
(deserializer) depend on sema-core as a Cargo crate via
flake-crates/.

## .aski Definitions

Parse tree domains (generated by corec):
- `core/root.aski` — RootChild, ModuleDef, EnumDef, StructDef, etc.
- `core/expr.aski` — Expr, FieldInit
- `core/statement.aski` — Statement, Instance, Mutation, Loop, Iteration
- `core/body.aski` — Block, MethodBody, Param
- `core/type.aski` — TypeExpr, TypeApplication
- `core/trait.aski` — TraitDeclDef, TraitImplDef, MethodDef
- `core/pattern.aski` — MatchExpr, MatchArm, Pattern

Defined in src/lib.rs (bootstrap):
- TypeName, VariantName, FieldName, MethodName, TraitName, etc.
- Span

## How It Works

```
core/*.aski → corec → generated/sema_core.rs → lib.rs includes it
```

Run `corec core generated/sema_core.rs` to regenerate locally.

## The Pipeline

```
corec       — .aski → Rust with rkyv derives (the tool)
aski-core   — grammar .aski + corec → Rust rkyv types (askicc↔askic)
sema-core   — parse tree .aski + corec → Rust rkyv types (this repo)
askicc      — uses aski-core types → rkyv dialect-data-tree
askic       — uses aski-core (input) + sema-core (output)
semac       — uses sema-core types only
```

## Rust Style

**No free functions — methods on types always.** `main` is
the only exception.
